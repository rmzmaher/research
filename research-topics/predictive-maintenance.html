Predictive Maintenance for Electrical Submersible Pumps

Predictive Analytics Using Machine Learning for Equipment Failure Forecasting

Introduction

Electrical Submersible Pumps (ESPs) are critical artificial lift systems that require significant maintenance resources and manpower. Maintenance is usually triggered by reactive process monitoring of multivariate sensor data. Understanding equipment status through machine learning enables predictive maintenance, avoiding operation downtime and extending equipment life.

ESPs are installed in many producing wells subject to harsh environments, pumping complex fluid mixtures that undergo changes in composition, pressure, and temperature over time. For assuring reliable fluid delivery, timely interventions are required in case of upcoming problems. This research focuses on deploying machine learning and data-driven models in the area of predictive pump maintenance.

Research Objective

Increase efficiency by reducing the time required to dismantle systems, inspect them, and perform failure analysis. The objective is achieved by applying Principal Component Analysis (PCA) as a dimensionality reduction technique, then pipelining its output with a model trained using Extreme Gradient Boosting (XGBoost) algorithm for future prediction of pre-workover events.

Key Innovation: The model can identify deeper functional relationships and longer-term trends inferred from historical data, providing alarms within seven days before actual failure events.

Data Gathering and Pre-Processing

Time series data collected from ESP sensors include:

Frequency Data

Pump frequency (Hz) indicating operational speed

Pressure Data

Pump discharge pressure (PDP), pump intake pressure (PIP), wellhead pressure (WHP)

Temperature Data

Wellhead temperature (WHT), motor temperature (MT)

Electrical Data

Motor current, casing head pressure (CHP)

ESP Failure Categories

Based on workover status sheets, pumps exhibit two main categories of electrical failures:

Motor Downhole Failures (MDHF)

Stator failure due to overheating, polymer deposition reducing heat dissipation

Electrical Downhole Failures (EDHF)

Cable insulation due to corrosion, material failure, overload conditions

Principal Component Analysis (PCA)

PCA is applied as an unsupervised dimensionality reduction technique. It reduces large dimensionality datasets into lower dimensions called principal components while preserving as much information as possible. This results in reducing the dimension of production parameters by making the most of the linear combinations.

PCA Calculation Steps

  1. Compute Covariance Matrix: Calculate relationships between contributing features
  2. Calculate Eigenvectors & Eigenvalues: Determine principal components of the data
  3. Select Principal Components: Choose components based on maximum variance criterion
  4. Transform Data: Project samples into new lower-dimensional subspace

PCA Loading Factors

The loading matrix represents correlations between original variables and principal components. Large loadings (positive or negative) indicate strong relationships. Most correlated parameters with the first principal component are:

  • Pump Frequency: -0.90 loading
  • Casing Head Pressure: -0.85 loading
  • Current: -0.84 loading
  • Motor Temperature: -0.79 loading
  • Wellhead Temperature: -0.63 loading

XGBoost Application

Extreme Gradient Boosting (XGBoost) is a tree-based ensemble model that combines the predictive abilities of multiple models. The final prediction is performed through the combination of these predictors. XGBoost builds trees sequentially, where each tree reduces the error of its predecessor while updating residual errors.

XGBoost Objective Function

At each iteration, the objective function combines loss function and regularization:

  • L(t) = Σ l(yi, pi + Ovalue) + (1/2)λ(Ovalue)²
  • l(yi, pi) = -[yi log(pi) + (1-yi) log(1-pi)]

Second-order Taylor approximation enables traditional optimization techniques in Euclidean space.

Model Training & Validation

The dataset is divided into training (70%), validation (15%), and testing (15%) sets. K-fold cross-validation (k=10) is applied to minimize the risk of selecting validation points with disproportionate difficulty.

Research Findings

Validation Set Performance:

  • Mean AUC = 0.95: The model demonstrates adequate performance and generalization capability
  • Consistent across folds: Low variance in performance across different validation sets
  • Robust to data distribution: Handles varying operational conditions effectively

Testing Set Performance:

  • Precision = 0.8: High precision minimizes false alarms (only 20% false positive rate)
  • Recall = 0.6: Not all 7 days before event show warnings, which is acceptable
  • Rationale: High precision preferred over high recall to minimize disruptive false alarms
  • Practical Impact: Operators receive reliable warnings with sufficient lead time

Hyperparameter Tuning

XGBoost hyperparameters were optimized using grid search across the following ranges:

  • max_depth: 2-10 (controls overfitting through tree depth)
  • min_child_weight: 1e-10 to 1e10 (minimum sum of weights in child node)
  • colsample_bytree: 0-1 (subsample ratio of columns per tree)
  • learning_rate: 0-0.1 (step size shrinkage for overfitting prevention)
  • gamma: 0-5 (minimum loss reduction for split)

Overfitting Prevention

Overfitting techniques were rigorously applied to ensure model generalization. K-fold cross-validation (k=10) detects the optimal stopping point and prevents fitting noise instead of underlying concepts.

Validation Strategy

Each model is trained on all subsets except one, tested on the held-out subset, and performance is averaged over all folds. The model with lowest estimated generalization error is selected and retrained on entire training set including validation data.

Practical Applications

  • Maintenance Scheduling: 7-day advance warning enables planned interventions
  • Cost Reduction: Minimizes unplanned downtime and emergency repairs
  • Resource Optimization: Better allocation of maintenance crews and equipment
  • Production Continuity: Maintains stable production through proactive maintenance
  • Equipment Lifespan: Early intervention prevents cascading failures

Future Directions

  • Transfer Learning: Applying models across different fields with minimal retraining
  • Explainable AI: Enhanced interpretability for engineering insights
  • Remaining Useful Life: Predicting time-to-failure in addition to failure probability
  • Digital Twin Integration: Combining with physics-based simulations
