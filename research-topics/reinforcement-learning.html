Reinforcement Learning for Energy Systems Optimization

Prescriptive Analytics Using Actor-Critic Methods for Production Optimization

Introduction

Reinforcement Learning (RL) represents a paradigm shift in production optimization for energy systems. This research applies model-free RL algorithms for optimal control of injection rates in enhanced oil recovery operations. The approach overcomes limitations of traditional optimization methods by learning optimal policies through environmental interactions without requiring full system description.

Figure 1: Agent-environment interaction in Reinforcement Learning showing the closed-loop control structure.

Why Reinforcement Learning?

RL is preferred for energy systems optimization because it:

  • Model-Free: Does not require full description of the process
  • Learning from Experience: Takes advantage of previous interactions to find optimal policies
  • Automated: Maximizes net present value without human interference
  • Adaptive: Handles dynamic reservoir conditions and uncertainty

Elements of Reinforcement Learning

Environment

Reservoir simulation model representing system behavior

State

Operating conditions: cumulative production, injection

Action

Control decisions: increase, decrease, maintain rates

Reward

Net present value maximization objective

Policy

Mapping from states to actions (learned strategy)

Value Function

Long-term return estimation from states

Application: Steam Injection Optimization

Steam flooding is a thermal oil recovery method where steam forms a condensing zone inside the reservoir. The heat of condensation reduces crude oil viscosity, facilitating its displacement. This research applies RL to optimize dynamic injection rates, addressing the challenge that no single injection rate value is optimal for all situations.

Comparison with Traditional Optimization

Traditional approaches have limitations:

  • Genetic Algorithms: Provide only single injection rate value (steady-state), 6% better than others but insufficient for dynamic problems
  • Model Predictive Control: Requires proxy models, unstable, suitable only for small horizons
  • Adjoint Methods: Require hard-coded gradients, difficult for compositional thermal models

Actor-Critic Architecture

The Advantage Actor-Critic (A2C) method combines value-based and policy-based approaches. It consists of two neural networks and the advantage function that calculates TD Error (Prediction Error).

Figure 4: Actor-Critic network architecture showing policy network (actor) and value network (critic) interaction.
Figure 5: Implementation of actor-critic architecture interacting with reservoir simulation environment (Eclipse 300).

Actor-Critic Components

  • Actor Network: Policy gradient algorithm that chooses actions at each time step
  • Critic Network: Evaluates Q-value providing feedback on how to adjust
  • Advantage Function: Captures the importance of an action compared to others for a given state

While the critic learns which states are better or worse, the actor uses these results to teach the agent to seek good states and avoid bad states.

Implementation Details

State Representation

The state captures the environment at its fundamental level, carrying information about process history:

St = [Cumulative Oil Production, Cumulative Steam Injection, Cumulative Water Production]

Action Space (Discrete)

Three possible actions with reference to previous timestep injection rate:

  • Action 0: Increase injection rate by 20 BPD
  • Action 1: Decrease injection rate by 20 BPD
  • Action 2: Maintain current injection rate

This discrete action space prevents dramatic changes in steam injection.

Reward Function

Net Present Value (NPV) at each timestep:

Rt = NPVt = (Po·qo - Csteam·qs - Cwater·qw) / (1 + i)^((t-tref)/365)

Where: Po = $100/STB, Csteam = $12/STB, Cwater = $3/STB, i = 0.2 (discount factor)

Policy Update

The actor network represents parameterized policy πθ mapping states to actions:

  • πθ(s,a) = argmax(Softmax P[a|s,θ])
  • ∇θ J(θ) = ∇θ log πθ(a|s) V^πθ(St)
  • θt+1 = θt + α ∇θ J(θ)

Advantage Function

To reduce high variance from trajectory updates, the advantage function is used:

A(s,a) = Q(s,a) - V(s)

This captures the degree of importance an action has compared to others for a given state.

Reservoir Simulation Environment

The environment is a compositional thermal reservoir model with following specifications:

Rock Properties

  • Porosity: 0.3 (fraction)
  • Vertical permeability: 50% of horizontal
  • Thermal conductivity: 24 BTU/(ft-D-°F)
  • Rock compressibility: 5×10⁻⁴ psi⁻¹

Fluid Properties

  • Oil density: 60.68 lbm/ft³
  • Molecular weight: 600
  • Viscosity: 5,780 cp at 75°F
  • Thermal expansion: 3.8×10⁻⁴ °F⁻¹

Operating Conditions

  • Max BHP: 1,600 psia (injection)
  • Min BHP: 17 psia (production)
  • Max injection rate: 300 STB/D
  • Production horizon: 820 days

Research Findings

Advantages of RL Approach:

  • Model-Free Learning: No requirement for full system description or complex physics modeling
  • Dynamic Optimization: Continuous adaptation to changing reservoir conditions
  • NPV Maximization: Directly optimizes economic objective function
  • Autonomous Operation: Learns optimal policies without human intervention
  • Scalable: Extends to multi-agent systems for complex field operations

Comparison with Literature:

  • vs. Genetic Algorithms: Provides dynamic time-varying strategies instead of single values
  • vs. MPC: No proxy model required, suitable for long production horizons
  • vs. Adjoint Methods: No need for hard-coded gradients, handles compositional models
  • vs. SARSA (on-policy RL): Actor-critic evaluates and enhances policy continuously

Future Directions

  • Multi-Agent Extensions: Coordinated control of multiple injection/production wells
  • Physics-Informed RL: Integration with reservoir physics constraints
  • Transfer Learning: Applying learned policies across different reservoir types
  • Real-Time Deployment: Edge computing for field implementation
  • Uncertainty Quantification: Robust policies under geological uncertainty
  • Multi-Objective Optimization: Balancing production, cost, and environmental impact
